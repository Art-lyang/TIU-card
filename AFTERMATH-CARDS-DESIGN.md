# TERMINAL SESSION — 여파 카드(Aftermath Cards) 설계안

> 작성일: 2026-04-08 (v0.5 기준, 구현 준비 단계)

---

## 개요

**여파 카드**란 플레이어의 중대한 선택 이후 1~2일 뒤에 자동으로 등장하는 후속 카드입니다.
숫자 변동만으로 체감하기 어려운 "내 선택이 세계를 바꿨다"는 느낌을 서사적으로 전달합니다.

### 기존 시스템과의 차이

| 기존 | 여파 카드 |
|------|----------|
| 선택 → 즉시 스탯 변동 | 선택 → 1~2일 후 서사적 후속 등장 |
| 카드 req에 LOG만 사용 | req에 LOG + day 범위 조건 사용 |
| 플레이어가 "결과"를 수치로만 인지 | 결과가 서사로 다시 돌아옴 |

## 트리거 설계

### req 패턴

```js
{
  id: "AF-001",
  act: [2],
  req: function(s, g, logs) {
    // LOG-RES-012 해금 후 2~4일 뒤
    return logs.includes("LOG-RES-012") && s.day >= (logs._dayMap && logs._dayMap["LOG-RES-012"] || 0) + 2;
  },
  msg: "윤세진이 Blood Pit 샘플 분석 중간 보고를 올렸다. ...",
  ...
}
```

**문제**: 현재 logs는 단순 문자열 배열이므로 "언제 해금됐는지" 추적이 안 됨.

### 해결 방안 (2가지)

**방안 A: day 최소값 기반 (간단)**
- LOG 해금 day를 추적하지 않고, 대신 `dayMin` 필드로 "최소 day N 이후" 조건만 사용
- 장점: 기존 시스템 변경 없음
- 단점: 정확한 "해금 후 N일"은 불가, 대략적 타이밍만 보장

```js
{ id: "AF-001", act: [2], dayMin: 8,
  req: function(s,g,logs){ return s.day >= 8 && logs.includes("LOG-RES-012") },
  msg: "윤세진 중간 보고: Blood Pit 조직 분석 결과..." }
```

**방안 B: LOG 해금 day 기록 (정밀, 시스템 변경 필요)**
- tryUnlock 시 `{ id: "LOG-xxx", day: stats.day }` 형태로 기록
- 기존 ts_logs 배열을 객체 배열로 마이그레이션 필요
- 장점: 정확한 타이밍 제어
- 단점: 세이브 구조 변경, 기존 데이터 호환 처리 필요

### 권장: 방안 A (dayMin 기반)

기존 시스템 변경 없이 구현 가능. 대부분의 여파 카드는 특정 Act의 특정 day 범위에서만 등장하므로 충분.

## 구현 계획

### 파일 구조

- `data-cards-16.js` — `CARDS_AFTERMATH` 변수 (여파 카드 15~20장)
- app-utils.js의 CARDS 병합에 `.concat(CARDS_AFTERMATH)` 추가
- index.html에 `<script src="data-cards-16.js?v=8">` 추가

### 카드 구조

```js
var CARDS_AFTERMATH = [
  {
    id: "AF-001",
    act: [2],
    priority: "중",
    msg: "윤세진 연구원이 중간 보고서를 제출했다.\n\nBlood Pit 조직에서 예상 밖의 반응이 확인되었습니다.\n\n윤세진: \"세포 재생 속도가 비정상적입니다. 이 데이터가 있으면...\"",
    left: { label: "연구 계속", fx: { r: -1, t: 1 } },
    right: { label: "연구 중단", fx: { r: 0, o: 1 } },
    req: function(s, g, logs) {
      return s.day >= 8 && logs.includes("LOG-RES-012");
    },
    tag: "aftermath",  // 5일 쿨다운으로 여파 카드 간 간격 보장
    bg: "lab"
  },
  // ...
];
```

### 쿨다운 처리

여파 카드에 공통 태그 `"aftermath"` 부여 + 쿨다운 5일.
→ 하루에 여파 카드가 연속 등장하지 않도록 방지.

### drawCard 수정

없음. 기존 `tag` 쿨다운 시스템으로 자동 처리.
`_cd_map`에 `'aftermath': 5` 추가만 필요.

## 추후 확장 의견 (#3, #4, #5)

### #3 메타 목표 가시성
- 3회차부터 "이번 세션 목표" 힌트를 ORACLE 브리핑에 추가
- 예: "미확인 데이터 복구", "전원 생존", "ORACLE 순응도 유지"
- 기존 엔딩 시스템과 연동하여 달성/미달성 피드백

### #4 관계 시스템 심화
- 현재: 신뢰도 수치 → 이브닝 챗 티어 + 대사 분기
- 제안: 신뢰 이벤트 추가 (특정 신뢰 구간 돌파 시 1회성 카드 등장)
  - 예: 서하은 trust 70 돌파 → "서하은이 개인적인 이야기를 꺼냈다" 카드
  - 이벤트 카드에서 선택 → 추가 LOG 해금 or 신뢰 큰 변동

### #5 멀티 세션 보상

- 2회차: Observer 암시 + 이전 엔딩 참조 대사
- 3회차+: 유도 지수 UI + 메타 목표 + 히든 카드
- 세이브 간 공유 데이터: ts_endings, ts_sessions
