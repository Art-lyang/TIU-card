## 1. 핵심 컨셉

### 한 줄 요약
> ORACLE이 배정한 임무를 수행하는 Proxy Network 요원으로서, 카드 스와이프로 기지 운영 판단을 내리고, 텍스트 어드벤처로 현장 임무를 수행한다. 당신의 모든 선택은 자유의지인가, 유도된 복종인가.

### 디자인 필라 (3대 원칙)

1. **자발적 복종의 게임화**
   플레이어는 ORACLE의 지시를 "합리적 판단"으로 받아들이며 게임을 진행한다. 그러나 게임의 숨겨진 레이어에서는 Observer의 유도가 선택지 자체를 조율하고 있다. 플레이어가 이 구조를 인지하는 순간이 게임의 진짜 클라이맥스.

2. **두 겹의 플레이**
   기지(카드 스와이프) ↔ 현장(텍스트 어드벤처)이 교차하며 리듬을 만든다. 기지에서 내린 판단이 현장 미션의 조건을 바꾸고, 현장에서 얻은 정보가 기지 카드 풀을 변화시킨다.

3. **ORACLE의 목소리**
   UI 전체가 ORACLE의 인터페이스다. 메뉴, 알림, 경고, 미션 브리핑 — 모든 텍스트가 ORACLE의 임상적 어조로 전달된다. 플레이어는 "게임을 하는 것"이 아니라 "ORACLE 단말기를 조작하는 것"이다.

## 1.5 플레이어 캐릭터

### 코드네임: Pilehead / 본명: 이중철

- **국적**: 대한민국
- **전직**: 육군 특수부대 장교 (대위 추정)
- **전역 사유**: 불명예전역. 프로메테우스 관계자가 ORACLE 관련 인물을 감시하는 과정에서 이중철의 부대와 접촉. 해당 관계자가 대한민국 및 부대원을 조롱·모욕하는 언행을 보임. 이중철이 과격한 방식으로 저지 — 이것이 불명예전역의 직접 원인.
- **현재**: ORACLE Proxy Network 한국 지부의 **초대 지휘관**으로 부임

> **캐릭터 디자인 의도**: 전역의 원인이 이념 대립이 아닌 **개인적 분노**. ORACLE의 "프로메테우스 = 위협" 분류와 개인 감정이 우연히 일치하면서, 플레이어는 ORACLE의 판단에 의문 없이 동의하기 쉬운 구조가 된다 — 또 하나의 자발적 복종 경로.

### 기지: Proxy Network 한국 지부

- **위치**: 강원도 내 비공개 위치 (코드: BRANCH KR-INIT-001)
- **규모**: 소규모. 인원 10명 내외.
- **설립 배경**: Observer와 ORACLE이 한국의 방어 성공(97.3%)에 주목하여 설립.

### 인물 구성

#### Proxy Network 한국 지부 (네임드 4명 + 지휘관)

**부지휘관/정보분석 — 서하은**
ORACLE 데이터 해석 및 브리핑 정리 담당. 지휘관 부임 전 3개월간 ORACLE 지시만으로 기지를 단독 운영. Act 4에서 ORACLE의 데이터 불일치를 처음 지적 → 전출 명령. 전출 저지 or 수용에 따라 Act 4 전체가 분기.

**현장작전/전투 — 강도윤**
전직 해병대 부사관 출신. 이중철의 필드 파트너. 과묵하고 실용적. 서하은 전출 시 강도윤이 빈자리를 체감하는 대사가 등장.

**의료/과학 — 윤세진**
이변체 표본 분석, 부상 치료 담당. EV-Σ에 대한 독자적 관찰 기록. EV-Σ Phase 0→1 전환을 40% 지연시키는 억제제 개발. GI 낮은 루트에서 핵심 정보 제공자.

**통신/기술 — 임재혁**
ORACLE 단말기 유지보수, 통신 보안 담당. ORACLE 의존도 팀 내 최고. ORACLE 아키텍처 내 비공개 레이어(5계층 이상) 발견.

#### 프로메테우스 한국 작전팀

**마르쿠스 베버** (지휘관) — 독일어 억양. 차분하고 논리적. ORACLE의 한국 데이터 이용 사실과 봉쇄 성공률 31% 기여 증거 보유.
**닉 포스터** — 이중철의 불명예전역 사건 당사자. 기지 주변 활동.
**박소영** — 데이터 분석관. 서하은의 추천으로 합류하는 민간인. 서하은 전출 후 후임 가능.
**에이전트 강 (강지현)** — 게임 내 정체 미공개. Act 2부터 흔적으로만 존재. ORACLE도 프로메테우스도 아닌 제3의 관찰자.

## 2. 게임 구조

### 2.1 사이클 개요

```
┌─────────────────────────────────────────┐
│            하루(Day) = 1 사이클           │
│                                         │
│  ① 카드 스와이프 (Act별 3/5/6/7장)       │
│  ② 이브닝 챗 (신뢰도 기반, 1인 랜덤)    │
│  ③ 현장 미션 (카드 선택에 의해 트리거)   │
│  ④ 일일 정보 수신 — 뉴스 헤드라인       │
│  ⑤ 보상 선택 (3개 중 택1)              │
│  ⑥ 일일 감쇠 적용 → 다음 날            │
└─────────────────────────────────────────┘
```

### 2.2 4-Act 구조 및 전환

4막 구조. Act 1(프롤로그) 이후, Act 2→3→4 전환은 **일수 기반 무조건 전환** + **조건에 따른 4루트 분기**.

#### Act 1: 프롤로그 (배치 前)
- ORACLE 단말기 첫 접속, 세계관 소개, 간부진 자기소개 이브닝 챗
- 고정 카드 풀 (프롤로그 전용, ~10장)
- 하루 카드: 3장 / 감쇠: 없음 / 이변체 조우: 없음
- 기간: 2~3일 (고정). 완료 후 Act 2로 즉시 전환.
- 목적: 게임 메카닉 학습 + 세계관 몰입 시작

#### Act 2: 부임 (Day 1~10)
- 기지 세팅, ORACLE 지시에 따른 정보 수집
- 프로메테우스 첫 감지, 이변체 첫 조우
- 하루 카드: 5장 / 감쇠: 없음

#### Act 3: 충돌 (Day 11~25)
- 프로메테우스 대립 본격화
- 서하은의 데이터 불일치 발견 → 전출 아크 준비
- 하루 카드: 6장 / 감쇠: c-1, r-1

#### Act 4: 선택 (Day 26+)
- 유도 지수에 따른 완전 분기
- 서하은 전출/잔류에 따른 정보 접근 차이
- 하루 카드: 7장 / 감쇠: c-2, r-2, t-1 / 진입 시 전체 -3

#### Act 전환 시스템 (Act 2→3→4)

Day ≥ 10 (Act2→3) 또는 Day ≥ 25 (Act3→4) 도달 시 **무조건 전환**:

| 루트 | Act2→3 조건 | Act3→4 조건 | 페널티 | 전용 카드 |
|------|------------|------------|--------|---------|
| **A 정상** | 프로메테우스 접촉 + 미션 완료 | 연쇄 완료 + 프로메 미션 | 0 | 없음 |
| **B 경험 부족** | 프로메테우스만 접촉 | 연쇄만 완료 | 전체 -2 | CT-001~002 / CT-008 |
| **C 정보 부재** | 미션만 완료 | 프로메 미션만 | 전체 -2 | CT-003~004 / CT-009 |
| **D 강제** | 둘 다 안 함 | 둘 다 안 함 | 전체 -4 | CT-005~007 / CT-010~011 |

Act 4 진입 시 추가 전체 -3 페널티.

### 2.3 기지 운영 (카드 스와이프 레이어)

ORACLE이 보고서/요청/상황 카드를 제시 → 좌/우 스와이프로 판단 → 4대 스탯 + GI 변동.

#### 4대 스탯

| 스탯 | 설명 | 시작값 | 0일 때 |
|------|------|--------|--------|
| **봉쇄** | 봉쇄선 유지도 | 50 | 게임 오버 |
| **자원** | 식량, 의약품, 장비 | 65 | 게임 오버 |
| **신뢰** | 기지 인원 신뢰도 | 50 | 게임 오버 |
| **평가** | ORACLE의 지휘관 평가 | 40 | 게임 오버 |

봉쇄 100 도달 시 `GRANT EXPIRED — UPON_FULL_ESTABLISHMENT` → 게임 종료.

#### 자원 리스크 시스템

자원(r) ≥ 2 획득 시 **20% 확률로 실패**. 실패 시 해당 카드의 r만 0으로 변경, 다른 스탯은 정상 적용.

**빨간 토스트 화면 중앙 표시, 2.5초.** (v0.5: 하단 사이드에서 중앙으로 변경)

메시지 4종 랜덤: "물자 상태 불량", "운송 중 파손", "유통기한 초과", "오염 감지"

#### 스탯 프리뷰

카드 스와이프 중 게이지바에 예상 변동량 표시:
- 증가: 녹색 반투명 영역 확장 + 숫자 녹색
- 감소: 빨간색 영역 + 숫자 빨간색
- 손 떼면 즉시 원래로 복귀

### 2.4 이브닝 챗 시스템 (v0.5 신규)

하루 카드를 모두 소화한 뒤, 저녁 시간대 간부진과의 대화 이벤트. 기존 대화 시스템과 별개.

#### 구조

| 항목 | 내용 |
|------|------|
| 발동 조건 | 매일 저녁 (카드 처리 후, 뉴스 전) |
| 등장 인원 | 4인 간부진 (신뢰도 조건 충족 시 박소영 추가) |
| 1회 표시 | 당일 1인만 등장 (랜덤 또는 조건 기반) |
| 신뢰도 구간 | low(0~35) / mid(36~60) / high(61~80) / bond(81~100) |
| 대사 변형 | 구간별 완전히 다른 대사 (단순 어투 변화가 아닌 내용 변화) |

#### 신뢰도 구간별 어투 방향성

| 인물 | low | mid | high | bond |
|------|-----|-----|------|------|
| 강도윤 | 최소한 보고, 벽 세움 | 사무적이지만 성실 | 솔직한 의견, 개인 이야기 | 전우 신뢰, 감정 표현 |
| 서하은 | 업무적, 거리감 | 조심스럽지만 솔직 | 경계 내려놓고 의견 개진 | 진심, 동료로서 의지 |
| 윤세진 | 형식적 보고 | 연구 보고 중심 | 개인 연구 고민 공유 | 감염 두려움, 솔직한 위기감 |
| 임재혁 | 기술적 사실만 | ORACLE 의존도 높은 보고 | ORACLE 의심 시작 | ORACLE 아래 레이어 발견 공유 |

#### 파일 분리

- `data-evening-trust-1.js` — `getTrustTier` 함수 + 강도윤 신뢰도별 대사 전체
- `data-evening-trust-2.js` — 서하은 신뢰도별 대사 전체
- `data-evening-trust-3.js` — 윤세진 · 임재혁 · 박소영 대사 + `EVENING_TRUST_LINES` 닫기

### 2.8 이변체 연쇄 시스템

5종 이변체 공통 3단계 구조:

```
첫 조우 (1회성, LOG 해금)
  → 관찰/연구 (LOG-020~024)
    → 제거 or 격리 결정 (미션 트리거)
```

## 3. 숨겨진 레이어: OBSERVER 시스템

### 3.1 유도 지수 (Guidance Index)

플레이어에게 **절대 표시되지 않는** 내부 변수.

- ORACLE 권장 방향 선택: +1~+2
- ORACLE 반항 선택: -1~-3
- 독립적 정보 수집: -3~-6

### 3.2 뉴스 헤드라인

하루 종료 시 1~3줄 표시. GI에 따른 필터링.

| GI 구간 | 필터링 |
|---------|--------|
| 낮음 | 다양한 뉴스 + 글리치 헤드라인 |
| 중간 | ORACLE에 불리한 뉴스 감소 |
| 높음 | 성과 보고로 변질 |

### 3.3 ORACLE 한줄평

GI 구간별 4종 × 4등급 = **16종 랜덤** 메시지.

| 구간 | 예시 |
|------|------|
| 고GI | "한국 지부 성과 상위권", "추가 권한 부여 검토 중" |
| 중GI | "운영 안정. 일부 비표준 패턴 감지" |
| 저GI | "독자적 의사결정 패턴 감지. 분석 중" |
| 극저GI | "지휘관 신뢰 지표 하락 중. 재평가 예정" |

## 6. 회차 플레이 시스템

### 6.1 인유니버스 근거

Observer가 ORACLE에 부여한 임시 능력 (GRANT). 플레이어가 경험하는 "회차"는 ORACLE이 지부 운영의 최적 경로를 탐색하는 과정.

### 6.2 부팅 시퀀스

| | 1회차 | 2회차+ |
|---|------|--------|
| GRANT | `GRANT: ACTIVE — TEMPORARY ACCESS` (시안) | `GRANT: ACTIVE — RENEWAL DETECTED` (시안) |
| 세션 | `TERMINAL SESSION — INITIATING` | `[OBSERVER: SESSION RESUMED]` (앰버) |
| 인사 | `WELCOME, COMMANDER.` | `WELCOME BACK, COMMANDER.` |
| 버튼 | `[ 세션 1 시작 ]` | `[ 세션 N 시작 ]` |

### 6.3 게임오버

```
─── SESSION #N TERMINATED ───
(사유 + 스탯 요약)
GRANT: ACTIVE — RENEWAL AVAILABLE
```

### 6.4 튜토리얼

3단계 (세계관 → 임무 → 스탯). 2회차 이상 시 첫 페이지에 `[ 튜토리얼 건너뛰기 ]` 버튼 표시.

### 6.5 회차별 해금

| 회차 | 해금 요소 | 상태 |
|------|-----------|------|
| 2회차 | ORACLE 잔상 로그, 튜토리얼 스킵, Observer 부팅 메시지 | ✅ |
| 3회차 | 이름 없는 게이지 UI (유도지수 표시) | 🔲 미구현 |
| 4회차+ | OBSERVER ACCESS 잔향 이벤트, F엔딩 조건 해금 | 🔲 미구현 |
