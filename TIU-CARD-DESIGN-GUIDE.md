# TERMINAL SESSION — 카드 설계 가이드

## 카드 구조
```js
{ 
  id: "C-XXX",           // 고유 ID
  priority: "상/중/하",   // 카드 상단 표시
  tag: "spec-011",       // 이변체 쿨다운 태그 (선택)
  req: (s,g,logs) => ..., // 출현 조건 (선택)
  msg: "카드 본문",       // \n으로 줄바꿈
  left: { label: "선택지", fx: { c:0, r:0, t:0, o:0 }, g: 0, mission: "M-XXX", trust: 10 },
  right: { label: "선택지", fx: { c:0, r:0, t:0, o:0 }, g: 0 },
}
```

## fx 값 가이드
- 각 값은 ×5 배율 적용됨 (1 = 실제 +5)
- 권장 범위: -3 ~ +3
- g(유도지수): ORACLE 따르면 +, 독립 행동 -

## 연쇄 카드 (CHAINS)
```js
CHAINS["CH-XXX"] = {
  name: "이름",
  trigger: "C-XXX-left/right",  // 트리거 카드ID + 방향
  cards: [ { id:"CH-XXX-1", ... }, ... ]
}
```
현재 6개: CH-001(서하은 조사), CH-002(봉쇄 위기), CH-003(ORACLE 시험),
CH-004(탈북자), CH-005(프로메테우스 접선), CH-006(서하은 마지막 밤)
