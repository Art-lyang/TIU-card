# TERMINAL SESSION — Act 구조 설계서

## 설계 원칙 (확정)

- **전환 기준**: 이벤트 기반 (특정 카드/미션 클리어 시 전환)
- **카드 풀**: 교체형 (일부 카드 퇴장 + 신규 카드 등장)
- **전환 알림**: ORACLE 브리핑 화면 (명시적 전환)

---

## 1. Act 개요

| Act | 서사 | 톤 | 예상 카드 수 |
|-----|------|----|-------------|
| **Act 1: 부임** | 지부 세팅, ORACLE 신뢰 구축 | 안정, 일상, 관리 시뮬 | ~30장 |
| **Act 2: 충돌** | 프로메테우스 대면, 균열 시작 | 긴장, 의심, 정보전 | ~35장 |
| **Act 3: 선택** | GI 기반 분기, 엔딩 수렴 | 위기, 선택의 무게 | ~25장 |

총 카드 풀: ~90장 (현재 86장 + 조정 필요)

---

## 2. Act 전환 트리거

### Act 1 → Act 2 전환

**필수 조건 (AND):**
1. `day >= 10` (최소 플레이 시간 보장)
2. 프로메테우스 첫 접촉 카드 클리어 (C-006 또는 C-011)
3. 현장 임무 최소 1회 완료 (M-001 / M-002 / M-003 중 아무거나)

**설계 의도**: 플레이어가 "기지 운영에 익숙해졌고, 외부 위협이 실재한다"는 것을 체감한 직후 전환. Day 기준은 하한선이지 자동 전환이 아님 — 이벤트를 안 밟으면 Act 1에 머무름.

**전환 연출**: 하루 종료 시 뉴스 대신 ORACLE 브리핑 화면 삽입.

```
[ORACLE BRIEFING — PRIORITY: ELEVATED]

지난 {day}일간의 운영 데이터를 분석했습니다.

봉쇄 구역 안정성: {c}%
자원 잔량: {r}%
인원 신뢰도: {t}%

평가: 초기 안정화 단계 완료.

그러나 — 새로운 변수가 감지되었습니다.
프로메테우스 활동 징후가 한국 내에서 증가하고 있습니다.
기존 운영 프로토콜을 재조정합니다.

[ENTER를 눌러 계속]
```

### Act 2 → Act 3 전환

**필수 조건 (AND):**
1. `day >= 25` (최소 Act 2 플레이 시간)
2. 연쇄 카드 최소 1개 완료 (CH-001 / CH-002 / CH-003 중)
3. 프로메테우스 관련 미션 완료 (M-003 "미분류 흔적 추적" 또는 Act 2 신규 프로메테우스 미션)

**설계 의도**: "ORACLE에 대한 의심이 최초로 형성된 직후" 전환. 연쇄 카드가 서사적 전환점 역할.

**전환 연출**: ORACLE 브리핑이 평소와 다르게 표시됨 — 텍스트 글리치 삽입.

```
[ORACLE BRIEFING — PRIORITY: CR█TICAL]

지부 운영 {day}일차 종합 보고.

봉쇄 구역 안정성: {c}%
자원 잔량: {r}%
인원 신뢰도: {t}%

경고: 프로메테우스 세력의 활동이 직접적 위협 수준에 도달했습니다.
지부 존속을 위한 결정적 조치가 필요합니다.

지휘관의 판단이 ██종 결과를 결정합니다.

[ENTER를 눌러 계속]
```

GI가 30 이하인 경우 추가 글리치 라인:
```
[...]
[관측 데이터 무결성: ██.█% — 재검증 필요]
```

## 3. 카드 풀 배분 (교체형)

### 3.1 카드 태그 시스템

모든 카드에 `act` 필드를 추가하여 소속을 정의:

```js
{
  id: "C-001",
  act: [1],          // Act 1 전용
  // ...
}
{
  id: "C-009",
  act: [1, 2],       // Act 1~2 공통
  // ...
}
{
  id: "C-050",
  act: [2, 3],       // Act 2~3 공통
  // ...
}
```

### 3.2 카드 풀 구성 원칙

| 카드 유형 | Act 1 | Act 2 | Act 3 |
|-----------|-------|-------|-------|
| **공통 (Core)** | ✅ | ✅ | ✅ |
| **Act 전용** | ✅ | ✅ | ✅ |
| **전이 (Crossover)** | Act 1~2 공유 | Act 2~3 공유 | — |
| **조건부** | 일부 | 일부 | 대다수 |

#### Core 카드 (전 Act 공통, ~15장)
변하지 않는 기지 운영 기본 카드. 보급, 순찰, 시설 유지 등.
- 톤은 바뀌지 않지만 스탯 효과(fx)가 Act에 따라 **가중**될 수 있음
- 예: 같은 보급 카드라도 Act 3에서는 자원 소모가 더 큼 (위기 상황)

#### Act 1 전용 카드 (~15장)
- [일상] 기지 초기 세팅, 요원 배치, 기본 순찰
- [인물] 캐릭터 첫 만남 이벤트
- [이변체] 소규모 이변체 활동 (SPEC-012 징후, 미분류 신호)
- [ORACLE] 초기 시스템 제안 (낮은 g 변동)
- Act 2 진입 시 **퇴장** — 더 이상 덱에 나타나지 않음

#### Act 2 전용 카드 (~20장)
- [프로메테우스] 정보전, 추적, 비공식 접촉
- [인물] 서하은 불일치 발견, 강도윤 독자 행동, 임재혁 기술 딜레마
- [ORACLE] 자동화 확대 요구, 권한 변경 (높은 g 변동)
- [외부] 소바리/DPRK/필라 뉴스 수신
- [이변체] SPEC-011 본격 활동, 미분류 이변체
- Act 1 전용 카드가 퇴장하고 이 카드들이 등장

#### Act 3 전용 카드 (~10장)
- [위기] 복합 긴급 상황 (다중 스탯 대폭 변동)
- [프로메테우스] 최종 대면, 진실 접촉
- [ORACLE] 최종 제안 ("결정적 타격" / "완전 자동화")
- [인물] 인물별 최종 이벤트 (신뢰도 분기)

#### 전이 카드 (~6~8장)
- Act 1→2: 프로메테우스 관련 기초 카드가 Act 2에서도 유지
- Act 2→3: 핵심 갈등 카드가 Act 3까지 이어짐

### 3.3 카드 풀 전환 메카닉 (구현)

```js
// 현재 Act에 따라 활성 카드 풀 필터링
function getActivePool(cards, currentAct, stats, gi, logs) {
  return cards.filter(card => {
    // Act 소속 체크
    if (!card.act.includes(currentAct)) return false;
    // 조건부 카드 체크
    if (card.req && !card.req(stats, gi, logs)) return false;
    return true;
  });
}
```

## 4. Act별 시스템 변화

### 4.1 난이도 곡선

| 요소 | Act 1 | Act 2 | Act 3 |
|------|-------|-------|-------|
| **하루 카드 수** | 5장 | 6장 | 7장 |
| **스탯 감쇠** | 없음 | 매일 c-2, r-2 | 매일 c-3, r-3, t-1 |
| **이변체 조우 확률** | 15% | 30% | 50% |
| **프로메테우스 카드 비율** | 5% | 20% | 35% |
| **ORACLE 자동 제안 빈도** | 낮음 | 중간 | 높음 (GI 높을수록 증가) |

### 4.2 뉴스 헤드라인 변화

| Act | 뉴스 톤 | 예시 |
|-----|---------|------|
| **1** | 일상적, 먼 위협 | "봉쇄 구역 인근 교통 통제 연장" |
| **2** | 불안, 의문 제기 | "봉쇄 구역 확대 — 당국 '통제 가능' 반복" |
| **3** | 위기, 공포 | "서울 동부 민간인 대피 시작", "통신 두절 지역 확대" |

### 4.3 대화 톤 변화

| Act | 인물 대화 톤 |
|-----|-------------|
| **1** | 소개, 협력적, 긍정적 |
| **2** | 의심, 갈등, 개인사 노출 |
| **3** | 결단, 충성 시험, 최후 통첩 |

대화 이벤트에도 `act` 필드 추가하여 Act별 다른 대화 풀 사용:
```js
DIALOGUES = [
  { id: "DLG-001", char: "서하은", act: [1], ... },  // Act 1 전용
  { id: "DLG-010", char: "서하은", act: [2], ... },  // Act 2 전용 (긴장)
  { id: "DLG-018", char: "서하은", act: [3], ... },  // Act 3 전용 (결단)
]
```

## 5. Act별 미션 배분

| 미션 | Act | 트리거 |
|------|-----|--------|
| M-001 Blood Pit 샘플 채취 | 1 | C-010 오른쪽 |
| M-002 Shell Talker 대응 | 1~2 | C-003 오른쪽 |
| M-003 미분류 흔적 추적 | 1~2 | C-019 오른쪽 |
| **M-004 (신규) 프로메테우스 은신처** | 2 | Act 2 신규 카드 |
| **M-005 (신규) 봉쇄선 붕괴 지점** | 2~3 | Act 2~3 위기 카드 |
| **M-006 (신규) 최종 작전** | 3 | Act 3 전환점 카드 |

### 신규 미션 개요

**M-004 "프로메테우스 은신처"** (Act 2)
- M-003에서 발견한 흔적 후속
- 에이전트 강 간접 대면 (목소리만 / 통신만)
- 프로메테우스가 "적이 아닐 수 있다"는 첫 암시
- GI에 따라 분기: 높으면 ORACLE 보고 → 습격, 낮으면 대화 시도

**M-005 "봉쇄선 붕괴 지점"** (Act 2~3)
- 복수 이변체 동시 활동
- 생존 호러 톤 ("봉쇄선까지" 단편과 공명)
- 요원 사상자 발생 가능 → t 대폭 하락 또는 상승

**M-006 "최종 작전"** (Act 3)
- GI 높음: ORACLE이 지시하는 "프로메테우스 거점 타격"
- GI 낮음: 프로메테우스와의 합류 가능성
- 엔딩 직결 미션

## 6. Act별 연쇄 카드 배분

| 연쇄 | Act | 서사 역할 |
|------|-----|----------|
| CH-001 "서하은의 조사" | 1~2 | Act 2 전환 씨앗 |
| CH-002 "봉쇄 위기" | 2 | 이변체 위협 확대 |
| CH-003 "ORACLE의 시험" | 2 | GI 핵심 분기점 |
| **CH-004 (신규) "닉 포스터의 제안"** | 2~3 | 프로메테우스 접촉 |
| **CH-005 (신규) "관측중지"** | 3 | Observer 암시 클라이맥스 |
| **CH-006 (신규) "최후의 세션"** | 3 | F엔딩 루트 진입 |

### 신규 연쇄 개요

**CH-004 "닉 포스터의 제안"** (Act 2~3)
- 트리거: 프로메테우스 관련 카드에서 비공식 접촉 선택
- 포스터가 직접 메시지 전달 → "ORACLE이 보여주지 않는 것이 있다"
- 선택: 무시(g+) / 청취(g-)

**CH-005 "관측중지"** (Act 3)
- 트리거: GI ≤ 25 + LOG-009 해금
- ORACLE 데이터 스트림에서 OBSERVER ACCESS 흔적 포착
- 추적 시도 → 갑자기 세션 끊김 → "관측중지" 메시지
- 재접속 시 해당 로그가 삭제되어 있음

**CH-006 "최후의 세션"** (Act 3)
- 트리거: GI ≤ 15 + CH-005 완료 + 특정 로그 조합
- GRANT 로그 조각 수집 → 해독 시도
- F엔딩 직결

## 7. 엔딩 접근 조건 (Act 3 기준)

| 엔딩 | 접근 조건 |
|------|----------|
| **A. 완전한 도구** | GI ≥ 70 + Act 3 도달 + M-006(ORACLE 루트) 완료 |
| **B. 각성** | GI ≤ 30 + 특정 로그 3개 이상 + CH-005 완료 |
| **C. 봉쇄 붕괴** | c ≤ 0 (어느 Act에서든) |
| **D. 반란** | t ≤ 0 + r ≥ 50 (어느 Act에서든) |
| **E. 소바리 루트** | 소바리 관련 카드 3회 + 특정 미션 완료 |
| **F. SESSION DENIED** | GI ≤ 15 + CH-006 완료 + GRANT 로그 해독 |

## 8. 세이브 데이터 확장

기존 세이브에 `act` 필드 추가:

```js
saveData = {
  stats: { c, r, t, o, day },
  gi: 50,
  act: 1,                    // 현재 Act
  actFlags: {
    act1_prometheus_met: false,  // C-006/C-011 클리어 여부
    act1_mission_done: false,    // 현장 임무 1회 완료 여부
    act2_chain_done: false,      // 연쇄 카드 1개 완료 여부
    act2_prometheus_mission: false, // 프로메테우스 미션 완료 여부
  },
  usedCards: [],              // 이미 나온 카드 (중복 방지)
  retiredCards: [],           // 퇴장한 카드 (Act 전환 시)
  logs: [...],
  endings: [...],
  sessions: 0,
}
```

## 9. 구현 우선순위

1. **카드 태깅** — 기존 86장에 `act` 필드 부여
2. **Act 상태 관리** — App.jsx에 act state + 전환 로직
3. **카드 풀 필터** — `getActivePool()` 함수
4. **ORACLE 브리핑 화면** — 전환 연출 컴포넌트
5. **난이도 곡선 적용** — Act별 카드 수, 감쇠, 확률
6. **대화/뉴스 Act 태깅** — 기존 콘텐츠 분류
7. **신규 콘텐츠** — 신규 미션 3개 + 연쇄 3개 + Act 3 전용 카드
